# goit-algo2-hw-06

Домашнє завдання до модуля **«Робота з даними»**.

## Структура проєкту

- `task_1.py` — перевірка унікальності паролів за допомогою фільтра Блума.
- `task_2.py` — порівняння точного підрахунку унікальних IP-адрес із HyperLogLog.
- `requirements.txt` — залежності проєкту. Код використовує тільки стандартну бібліотеку Python.

## Завдання 1

У файлі `task_1.py` реалізовано:

- клас `BloomFilter`;
- методи `add` та `contains`;
- функцію `check_password_uniqueness`;
- обробку порожніх і некоректних значень;
- приклад використання з умови.

Запуск:

```bash
python task_1.py
```

Очікуваний приклад виводу:

```text
Пароль 'password123' — вже використаний.
Пароль 'newpassword' — унікальний.
Пароль 'admin123' — вже використаний.
Пароль 'guest' — унікальний.
```

## Завдання 2

У файлі `task_2.py` реалізовано:

- ліниве завантаження IP-адрес із лог-файлу;
- ігнорування некоректних рядків;
- точний підрахунок унікальних IP через `set`;
- наближений підрахунок через власну реалізацію `HyperLogLog`;
- порівняння часу виконання;
- виведення результатів у вигляді таблиці.

Файл `lms-stage-access.log` не додається до репозиторію та zip-архіву, тому що за умовою він завеликий для LMS.

Запуск, якщо лог лежить поруч зі скриптом:

```bash
python task_2.py
```

Запуск із передачею шляху до лог-файлу:

```bash
python task_2.py path/to/lms-stage-access.log
```

Приклад формату виводу:

```text
Результати порівняння:
                            Точний підрахунок    HyperLogLog
Унікальні елементи                    100000.0        99652.0
Час виконання (сек.)                    0.450000        0.100000

Похибка HyperLogLog: 0.35%
```

## Примітки

Фільтр Блума є ймовірнісною структурою даних: він може дати хибнопозитивний результат, але не дає хибнонегативних результатів.

HyperLogLog повертає наближену кількість унікальних елементів із малою похибкою та використовує значно менше памʼяті, ніж точний підрахунок через `set`.
